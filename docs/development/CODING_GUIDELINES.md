# Coding Guidelines

## Principles
- Correctness before cleverness.
- Explicit code over magic.
- Composition over inheritance.
- Fully typed public APIs.
- Domain never imports Infrastructure.

## Quality
- pytest required
- ruff clean
- mypy clean
- structured logging
- no TODO placeholders
- no wildcard imports

## Documentation
Public classes and modules require docstrings.
Update docs together with implementation.
